apache/vue/angular/react的配置使用时，有遇到一些小问题，记录下

2020年02月27日 1.4k 字 大概 6 分钟

apache/vue/angular/react的配置使用时，有遇到一些小问题，记录下

1.apache：

我是使用xampp软件的apache功能

http://localhost/默认打开的是：

但是，要访问的是本地服务器文件：

解决：进入Config中的Apache(httpd.conf),将LoadModule dir_module modules/mod_dir.so注释掉。

在以下框架进行安装前，安装好NodeJS。

2.Vue

1.安装vue-cli 脚手架

首先要安装Vue的脚手架，在cmd中输入npm install -g vue-cli安装。

2.新建一个vue项目模版

接着，进入要存放项目的文件夹，输入vue init webpack myProject创建项目模版，myProject是项目名称，可以自定义。

3.安装依赖包

在项目目录下输入cnpm install，安装部分完成了。

使用VScode时，打开项目文件夹，在终端中输入npm run dev即可进行运行项目。

4.在vue-cli项目中使用bootstrap

1.安装插件

npm install jquery –save

npm install bootstrap –save

npm install popper.js –save

2.如果cli版本小于v3：

配置webpack.base.conf.js

//在顶部添加
const webpack = require('webpack')
//在module.exports = {}末尾添加下面代码
module.exports = {
...
plugins: [
    new webpack.ProvidePlugin({
      $: "jquery",
      jQuery: "jquery"
    })
  ]
}

3.在Vue-cli v3的版本中需要使用下面的方法

在根目录中添加文件vue.config.js，然后写入下面配置

const webpack = require('webpack')
module.exports = {
  configureWebpack: {
    plugins: [
      new webpack.ProvidePlugin({
        $: "jquery",
        jQuery: "jquery"
      })
    ]
  }
}

4.main.js中添加

import $ from 'jquery'
import 'bootstrap/dist/css/bootstrap.min.css'   //我用的是3.0以下版本亲测
import 'bootstrap/dist/js/bootstrap.min'         //3.0以下版本亲测

3.Angular

1.安装angular cli

在cmd中输入npm install -g @angular/cli安装。

2.新建一个angular项目模版

进入要存放项目的文件夹，输入ng new myProject创建项目模版，myProject是项目名称，可以自定义。

这里我遇到到问题了，cmd一直无法下载完成模版文件,

解决：在Github里下载angular的quickstart模版可以进行代替。

使用VScode时，打开项目文件夹，在终端中输入npm start即可进行运行项目。

4.React

1.安装react-native-cli

在cmd中输入npm install -g react-native-cli安装。

2.新建一个react项目模版

进入要存放项目的文件夹，输入react-native init myProject创建项目模版，myProject是项目名称，可以自定义。

最后还是选择了yarn来安装：

npm install -g yarn react-native-cli                             
yarn config set registry https://registry.npm.taobao.org --global 
yarn config set disturl https://npm.taobao.org/dist --global
react-native init myProject

使用VScode时，打开项目文件夹，在终端中输入react-native start即可打开开发服务器。

2.5 在模拟器上调试项目

我用的mumu模拟器，打开cmd输入adb connect 127.0.0.1:7555连接模拟器，

打开模拟器，在项目安装目录下react-native run-android就可以将项目app安装模拟器

3.打包app

①生成签名密钥

你可以用keytool命令生成一个私有密钥。在Windows上keytool命令放在JDK的bin目录中（比如C:\Program Files\Java\jdkx.x.x_x\bin），你可能需要在命令行中先进入那个目录才能执行此命令。

keytool -genkey -v -keystore my-release-key.keystore -alias my-key-alias -keyalg RSA -keysize 2048 -validity 10000

这条命令会要求你输入密钥库（keystore）和对应密钥的密码，然后设置一些发行相关的信息。最后它会生成一个叫做my-release-key.keystore的密钥库文件。

②设置gradle变量

  1. 把my-release-key.keystore文件放到你工程中的android/app文件夹下。
  2. 编辑~/.gradle/gradle.properties（没有这个文件你就创建一个），添加如下的代码（注意把其中的****替换为相应密码）

注意：~表示用户目录，比如windows上可能是C:\Users\用户名，而mac上可能是/Users/用户名。

    MYAPP_RELEASE_STORE_FILE=my-release-key.keystore
    MYAPP_RELEASE_KEY_ALIAS=my-key-alias
    MYAPP_RELEASE_STORE_PASSWORD=*****
    MYAPP_RELEASE_KEY_PASSWORD=*****

③添加签名到项目的gradle配置文件

编辑项目目录下的android/app/build.gradle，添加如下的签名配置：

    ...                                  //'...'都是不需要修改或者添加的内容
  android {
    ...
    defaultConfig { ... }
    signingConfigs {
        release {                                              //需添加
            storeFile file(MYAPP_RELEASE_STORE_FILE)             
            storePassword MYAPP_RELEASE_STORE_PASSWORD
            keyAlias MYAPP_RELEASE_KEY_ALIAS
            keyPassword MYAPP_RELEASE_KEY_PASSWORD
        }
    }
    buildTypes {
        release {
            ...
            signingConfig signingConfigs.release              //需添加
        }
    }
  }
  ...

④生成发行APK包

只需在终端中运行以下命令：

cd android && ./gradlew assembleRelease

译注：cd android表示进入android目录（如果你已经在android目录中了那就不用输入了）。./gradlew assembleRelease在macOS、Linux或是windows的PowerShell环境中表示执行当前目录下的名为gradlew的脚本文件，且其运行参数为assembleRelease，注意这个./不可省略；而在windows的传统CMD命令行下则需要去掉./。

生成的APK文件位于android/app/build/outputs/apk/app-release.apk。

⑤测试应用的发行版本

在把发行版本提交到Play Store之前，你应该做一次最终测试。输入以下命令可以在设备上安装发行版本：

cd android && ./gradlew installRelease

5.打包app时遇到的问题

错误报告1：

Error:The SDK directory ‘xxxxxxxxxx’ does not exist. Please fix the ‘sdk.dir’ property in the local.properties file

解决：在react native项目的android目录下新建local.properties文件，文件内容为SDK路径：

windows下记得使用双斜杠。

错误报告2：

Failed to install the following Android SDK packages as some licences have not been accepted.
You have not accepted the license agreements of the following SDK components:

[Android SDK Build-Tools 24, Android SDK Platform 24].Before building your project, you need to accept the license agreements and complete the installation of the missing components using the Android Studio SDK Manager.

搜索了很多资料，都是说:

  1. 进入到 SDK 安装目录

  2. 在 bin 目录下执行如下命令 ：sdkmanager.bat –licenses

但是这没有解决问题，最后在评论区的回复中找到了正解：

    .\sdkmanager.bat "platforms;android-28"
