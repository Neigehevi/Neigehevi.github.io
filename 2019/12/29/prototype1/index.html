面对对象的一些知识点

2019年12月29日 576 字 大概 2 分钟

记录一些面对对象中原型继承和原型链的一些知识点。

面对对象

在学习了面对对象的原型继承和原型链，简单的写点总结，方便以后复习查看。

如果有地方写错了请给我留言QaQ。

原型

.prototype和.proto的区别

  1. 首先，.proto是对象的原型，所有对象都有它的原型（除了：object.prototype.proto=null）。
  2. 一般来说，只有函数才有prototype属性，prototype属性指向一个对象，这个对象也有它的原型，函数整体也是一个对象，所以函数也有.proto属性。
1 Cat=function(name){this.name=name;}
2 var hevi=new Cat            
3 hevi.Prototype             ------------->//undefined(函数才有Prototype属性)

原型继承和原型链

创建一个构造函数：

1   function Cat(name){
2     this.name=name;
3     }
4   Cat.prototype.say={
5     alert("这只猫叫"+this.name);
6     }

再创建一个子构造函数：

1    function Color(color){
2       Cat.call(this,name);        //用.call或者.apply调用Cat构造函数来使用this变量
3       this.color=color;
4       }

这个时候Color（）的原型链：

new Color()===>Color.prototype===>object.prototype===>null

函数Color（）要想使用函数Cat（）的方法.say则需要修改原型链为：

new Color()===>Color.prototype===>Cat.prototyepe===>object.prototype===>null

方法①：

1    Color.prototype=new Cat();
2    Color.prototype.constuctor=Color;

此时Color.prototype继承了函数Cat（）上的所有属性和方法。

方法②：

1    Color.prototype=Cat.prototyepe;
2    Color.prototype.constuctor=Color;

函数Cat（）的原型就是Cat.prototyepe，可以跳过Cat（）直接继承Cat.prototyepe.

以上两种方法的prototype都指向了同一个对象，当Color.prototype改变会导致Cat.prototyepe也发生改变，所以不建议使用。

方法③：

1    function Hevi();
2    Hevi.prototype=Cat.prototype;
3    Color.prototype=new Hevi();
4    Color.prototype.constuctor=Color;

创建一个空的构造函数Hevi（），当作中转，实现原型的继承,此时修改Color.prototype不影响Cat.prototype。

以上三种方法都要手动矫正Color.prototype.constuctor=Color。

方法④：

1    function extend0(Child,Parent){
2        var P=Parent.prototype;
3        var c=Child.prototype;
4        for(var i in p){
5           c[i]=p[i];
6        }
7    }

将父构造函数函数中的原型对象上的属性和方法一个一个的传递到子构造函数的原型对象上。

这种方法并没有改变子构造函数的constuctor属性。
